api.query_commons module¶

api.query_commons.fetch_data(schema, table, root_select_expression_dict, max_page_size, request, where='', args=None, many=True, enricher=None, sort=None, limit=True, **nested_expression_dict)[source]¶

Fetches data in an JSON-API conforming way.

Parameters:
  • schema – the marshmallow schema to use for serialization, provided by faftools: https://github.com/FAForever/faftools/tree/develop/faf/api
  • table – the table to select the data from (or any FROM expression, without the FROM)
  • root_select_expression_dict – a dictionary that maps API field names to select expressions
  • max_page_size – max number of items per page
  • request – the flask HTTP request
  • where – additional WHERE clauses, without the WHERE
  • args – arguments to use when building the SQL query (e.g. where="id = %(id)s", args=dict(id=id)
  • many – True for selecting many entries, False for single entries
  • enricher – an option function to apply to each item BEFORE it’s dumped using the schema
  • sort – order the query by given column name in asc order, prefix with ‘-‘ for desc order
  • nested_expression_dict – dict of nested objects to be found in select_expression_dict e.g. nested_expression_dict = {‘nest_atr_name’ : { ‘nest_atr_key’ : ‘nest_atr_value’}}

api.query_commons.get_order_by(sort_expression, valid_fields)[source]¶

Converts the sort_expression into an “order by” if all fields are in field_expression_dict Example usage:

sort_expression = 'likes,-timestamp'
field_expression_dict = {
    'id': 'map.uid',
    'timestamp': 'UNIX_TIMESTAMP(t.date)',
    'likes': 'feature.likes'
}

get_order_by(sort_expression, field_expression_dict)

Result:

"ORDER BY likes ASC, timestamp DESC"
Parameters:
  • sort_expression – a json-api conform sort expression (see example above)
  • valid_fields – a list of valid sort fields
Returns:

an MySQL conform ORDER BY string (see example above) or an empty string if sort_expression is None or

empty

api.query_commons.get_select_expressions(fields, field_expression_dict)[source]¶

Returns select expressions for all fields in field_expression_dict. Example usage:

fields = ['id', 'timestamp']
field_expression_dict = {
    'id': 'map.uid',
    'timestamp': 'UNIX_TIMESTAMP(t.date)',
    'likes': 'feature.likes'
}

get_select_expressions(fields, field_expression_dict)

Result:

"map.uid as id, UNIX_TIMESTAMP(t.date) as timestamp"
Parameters:fields – the list of fields to select from field_expression_dict. If None or empty, all fields will be

returned :param field_expression_dict: a dictionary mapping field names to select expressions. The expressions must not contain “as” clauses since they will be appended using the field (see example above) :return: a select expressions string (see example above)
