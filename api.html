api.achievements.achievements_update_multiple()[source]¶

Updates multiple achievements for the currently authenticated player.

Example Request:

POST /achievement/updateMultiple

{
  "updates": [
    {
        "achievement_id": string,
        "update_type": string,
        "steps": integer
    }
  ]
}

``updateType`` being one of "REVEAL", "INCREMENT", "UNLOCK" or "SET_STEPS_AT_LEAST"
``steps`` being mandatory only for update type `` INCREMENT`` and ``SET_STEPS_AT_LEAST``

Example Response:

HTTP/1.1 200 OK
Vary: Accept
Content-Type: text/javascript

{
  "updated_achievements": [
    {
      "achievement_id": string,
      "current_state": string,
      "current_steps": integer,
      "newly_unlocked": boolean,
    }
  ],
}
Status 200:No error

Ranked 1v1 Endpoints¶

api.ranked1v1.ranked1v1()[source]¶

Lists all ranked 1v1 players.

Example Request:

Default Values:

page[number]=1

page[size]=5000

GET /ranked1v1
Accept: application/vnd.api+json

Example Response:

HTTP/1.1 200 OK
Vary: Accept
Content-Type: text/javascript

{
  "data": [
    {
      "attributes": {
        "deviation": 48.4808,
        "id": "781",
        "is_active": true,
        "login": "Zock",
        "mean": 2475.69,
        "num_games": 1285,
        "ranking": 1,
        "rating": 2330,
        "won_games": 946
      },
      "id": "781",
      "type": "ranked1v1"
    },
    ...
  ]
}
Parameters:
  • page[number] (int) – The page number being requested (EX.: /ranked1v1?page[number]=2)
  • page[size] (int) – The total amount of players to grab by default (EX.: /ranked1v1?page[size]=10)
  • filter[isActive] (boolean) – Whether or not to filter active players or not (true or false) (EX.: /ranked1v1?filter[is_active]=true)
  • filter[player] (name) – Allows search functionality in the ranked1v1 endpoint based upon the players login name (EX.: /ranked1v1?filter[player]=Zock)
Status 200:

No error
